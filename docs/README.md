# CMS Documentation

Engineering standards for the Clinic Management System. Read these before adding
a feature — they exist so the codebase stays consistent as it grows.

- **[ARCHITECTURE.md](./ARCHITECTURE.md)** — principles, tech stack, repo layout,
  cross-cutting rules (identifiers, errors, auth, git), and the alignment backlog.
  **Start here.**
- **[frontend-standards.md](./frontend-standards.md)** — components, screen
  archetypes, forms, data layer, theme.
- **[backend-standards.md](./backend-standards.md)** — layering, API conventions,
  pagination, validation, database, migrations.

See [SETUP.md](../SETUP.md) for running the app locally.
